# СКУД — Система контроля и управления доступом на основе динамических QR-кодов

Веб-приложение для генерации и валидации динамических QR-кодов доступа. Разработано для СКУД (систем контроля и управления доступом).

**Стек технологий:**
- **Бэкенд:** Go (Gin framework) + SQLite
- **Фронтенд:** React 17
- **Криптография:** RSA-2048 подпись с SHA-256 (TOTP-стиль)

## Как это работает

### 1. Аутентификация пользователя

Пользователь входит в систему по логину и паролю. Бэкенд проверяет учетные данные в SQLite и выдает JWT-токен (24 часа действия).

**Тестовые учетные записи:**
| Логин    | Пароль    | ФИО               | Роль    |
|----------|-----------|-------------------|---------|
| admin    | admin123  | Администратор     | admin   |
| user1    | user123   | Иванов Иван       | user    |
| user2    | user123   | Петров Петр       | user    |

### 2. Генерация динамического QR-кода

После входа пользователь видит QR-код, который:

1. **Формирует данные:** `user_id:full_name:role:timestamp_interval:expiry_timestamp`
   - `timestamp_interval` — текущее время, округленное до 30-секундного интервала (TOTP-стиль)
   - `expiry_timestamp` — время истечения (+30 секунд)

2. **Подписывает RSA-ключом:** Данные хэшируются через SHA-256, подписываются приватным RSA-ключом (2048 бит)

3. **Формирует токен:** `данные|RSA-подпись`

4. **Сохраняет в БД:** QR-код записывается в таблицу `qr_codes`

5. **Отображает пользователю:** QR-код генерируется из полного токена и автоматически обновляется каждые 30 секунд

### 3. Валидация QR-кода (СКУД)

Считыватель отправляет токен на эндпоинт `/api/validate-qr`. Сервер:

1. **Проверяет RSA-подпись** — если подпись недействительна, доступ запрещен (QR-код подделан)
2. **Проверяет срок действия** — если QR-код истек (>30 секунд), доступ запрещен
3. **Проверяет повторное использование** — если QR-код уже был использован в этом интервале, доступ запрещен
4. **Проверяет существование пользователя** — если пользователь удален, доступ запрещен
5. **Логирует событие** — запись в таблицу `access_logs`

### 4. База данных SQLite

Три таблицы:

- **users** — пользователи системы (логин, хэш пароля bcrypt, ФИО, роль)
- **qr_codes** — история сгенерированных QR-кодов (токен, время выпуска/истечения, флаг использования)
- **access_logs** — журнал попыток доступа (токен, пользователь, статус, сообщение, время сканирования)

## API эндпоинты

| Метод | Путь                | Аутентификация | Описание                          |
|-------|---------------------|----------------|-----------------------------------|
| POST  | `/api/login`        | Нет            | Вход в систему                    |
| GET   | `/api/public-key`   | Нет            | Публичный RSA-ключ для считывателя|
| POST  | `/api/validate-qr`  | Нет            | Валидация QR-кода (СКУД)          |
| GET   | `/api/profile`      | JWT            | Профиль пользователя              |
| POST  | `/api/generate-qr`  | JWT            | Генерация нового QR-кода          |
| GET   | `/api/access-logs`  | JWT            | Журнал попыток доступа            |

## Запуск проекта

### Предварительные требования

- **Go** 1.21+
- **Node.js** 16+ (npm)
- Утилита `make` (опционально)

### 1. Запуск бэкенда (Go-сервер)

```bash
cd backend
go mod tidy
go run main.go
```

Сервер запустится на порту **8080**.

При первом запуске:
- Генерируется RSA-пара ключей (сохраняется в `backend/keys/`)
- Создается база данных SQLite (`backend/data/skud.db`) с тестовыми пользователями

**Сборка бинарника:**
```bash
cd backend
go build -o skud-server .
./skud-server
```

### 2. Запуск фронтенда (React)

В отдельном терминале:

```bash
npm install --legacy-peer-deps
NODE_OPTIONS=--openssl-legacy-provider npm start
```

Фронтенд запустится на порту **3000** и будет обращаться к бэкенду на `http://localhost:8080`.

### 3. Открытие в браузере

Перейдите на [http://localhost:3000](http://localhost:3000) — откроется страница входа.

## Формат QR-кода для СКУД-считывателя

Токен QR-кода имеет формат:

```
user_id:full_name:role:timestamp_interval:expiry_timestamp|base64_rsa_signature
```

Пример:
```
1:Администратор:admin:1784271600:1784271630|iEwDvivql04CqogEBEUdK7NcY1...
```

Считыватель должен отправить этот токен на `POST /api/validate-qr` для проверки доступа.

## Безопасность

- Пароли хранятся в виде bcrypt-хэшей
- JWT-токены с ограниченным сроком действия (24 часа)
- RSA-2048 подпись данных QR-кода (предотвращает подделку)
- QR-код действителен только 30 секунд (TOTP-стиль)
- QR-код нельзя использовать повторно в течение интервала
- CORS настроен для работы с произвольных источников
