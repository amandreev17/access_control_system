# План курсовой работы: 

## Разработка мобильного приложения с динамическим QR-кодом

## 1. Серверная часть (Backend) на языке Go

Разработка модуля бизнес-логики и защищенного API.

* **Аутентификация пользователей:**
  * Реализация механизма входа в систему по связке `логин/пароль`.
* **Работа с базой данных:**
  * Поднятие и интеграция СУБД.
  * Создание схемы данных для хранения учетных записей пользователей и метаданных QR-кодов.
* **Обеспечение безопасности:**
  * Хранение паролей в захэшированном виде.
  * Генерация динамических QR-кодов с интервалом обновления **каждые 10 секунд** (TOTP-подобный механизм).
  * Валидация QR-кода на серверной стороне перед выполнением критических операций.

**Результат этапа:** Готовая Go-библиотека для деплоя.


## 2. Клиентская часть (Frontend) на Android Studio

Разработка нативного мобильного интерфейса.

* **Структура экранов (Activity/Fragment):**
  * **Экран входа:** Поля для ввода логина и пароля, кнопка отправки данных на сервер.
  * **Личный кабинет:** Отображение информации о пользователе и актуального QR-кода.
* **Взаимодействие с сервером:**
  * Отправка авторизационных данных и получение токена.
  * Периодический запрос (каждые 10 секунд) для получения свежего QR-кода.
* **UX/UI требования:**
  * Минималистичный дизайн.
  * Отображение таймера обратного отсчета до следующей смены QR-кода.



## 3. Связь и интеграция

* Клиент получает QR-код в виде текстовой строки или изображения (base64) от Go-сервера.
* Сервер валидирует предъявленный QR-код, сверяя его с текущим временным слотом (Epoch time / 10).
